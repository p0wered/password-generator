# Пароль по названию сервиса

Веб-приложение на [Vite](https://vite.dev/) + [React](https://react.dev/) + TypeScript. По **секретной фразе**, названию сервиса и выбранной длине строится **один и тот же** пароль каждый раз — детерминированное отображение в читаемый набор символов.

## Зачем это нужно

Достаточно помнить секретную фразу и названия сервисов; при потере сохранённого пароля можно снова ввести те же данные и получить ту же строку.

## Как это устроено

- Стартовое состояние: **HMAC-SHA256** (Web Crypto), ключ — UTF-8 секретной фразы, сообщение — UTF-8 названия сервиса.
- Если символов пароля нужно больше, чем байт в одном блоке, состояние **наращивается** повторными SHA-256 от предыдущего хэша и суффикса счётчика (`:0`, `:1`, …).
- Байты по очереди отображаются в фиксированный алфавит: латиница, цифры и ограниченный набор символов.

Ничего не уходит на сервер: логика целиком на клиенте.

## Важно про безопасность

- **Секретную фразу** никому не сообщайте и не храните в открытом виде там, где к ней могут добраться другие. Без неё по одному только названию сервиса восстановить ваш пароль в этой схеме нельзя (для постороннего это выглядит как случайная строка).
- Регистр букв и **пробелы** в обеих строках учитываются.
- Смена **длины** пароля меняет всю строку целиком.
- Это не замена менеджеру паролей с защищённым хранилищем: при компрометации секрета все производные пароли для всех сервисов нужно считать скомпрометированными.

## Запуск

Требуется Node.js с npm.

```bash
npm install
npm run dev
```

Сборка для продакшена:

```bash
npm run build
npm run preview
```

## Стек

Vite 8, React 19, TypeScript, **Tailwind CSS v4** (`@tailwindcss/vite`), иконки [Lucide](https://lucide.dev/).
